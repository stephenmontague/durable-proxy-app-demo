Durable Cloud ↔ Edge Proxy
The Problem

Most everyone's in the Cloud today.

But plenty of critical software still runs only on local networks and on-prem servers — with very limited connection to the outside world.

  • Behind firewalls and inside VPCs — factories, hospitals, warehouses, utilities.
  • No inbound connections allowed in — and outbound is locked down to a trusted few.
  • Yet your cloud app still needs to reach them, and hear back — reliably.
How I Know About It

I've dealt with this first hand.

  • I inherited a legacy app that only supported TCP — and only in one direction.
  • It crashed constantly, and outbound HTTP failed so often my hand-rolled retries never really worked.
  • Every change had to be re-packaged, shipped to the client, and restarted by them inside their network or VPC.
  • It needed tweaking — sometimes a little, sometimes a lot — for every client that used it.
  • And once it was installed on-prem: zero visibility. We were flying blind.
How It Works
One proxy per site. Egress-only. Temporal makes every hop durable and de-duplicated.
Out Of The Box

What it solves on day one.

Egress-only security

No inbound ports at the edge. The proxy only ever dials out.

Durable & de-duped

Exactly-once-style delivery on Temporal. Survives outages; duplicates collapse to one.

Multi-protocol

HTTP, TCP (framed/MLLP, persistent sockets), FTP — out of the box.

Any format

JSON, XML, raw passthrough — chosen per message type.

Runtime config

Message catalog & routing are live state — hot-applied, no redeploys.

Remote ops

Enable / disable / restart over egress. Operator UI & full Temporal visibility.
